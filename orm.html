¿Que es un ORM?

Es una técnica/herramienta de software que permite interactuar con una base de datos relacional usando el paradigma de programación orientada a objetos

¿Para qué sirve un ORM?

Sirve para simplificar la interacción con la base de datos, permitiendo utilizar comandos de programación orientada a objetos en lugar de escribir consultas SQL manualmente, sus tres principales funciones son:

  • Hablar con la base de datos sin aprender su idioma: Podemos seguir usando el mismo lenguaje de programación (python, Javascript o java) y no se presentaran errores de sintaxis de SQL.
  • Guardar información como si fueran "Fichas":(Objetos) al momento de ingresar la información o el código, automáticamente se va guardando la información en filas tipo excel y el ORM lo organiza en tablas.
  • Cambiar de "almacén" sin desordenar nada Permite cambiar de base de datos sin necesidad de modificar el código, ya que el ORM se encarga de traducir las operaciones a la sintaxis específica de cada base de datos.

¿En que casos se usa un ORM?

Se usa en cualquier aplicación que se necesite guardar datos (Usuarios, productos, mensajes, etc) de forma organizada, ejemplo:

  • Crear una aplicación web o móvil
  • Modelos de datos complejos
  • Portabilidad de BD
  • Foco en el código
