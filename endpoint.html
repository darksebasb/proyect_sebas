¿Que es un endpoint?

Es una dirección específica (parecida a una URL de una página web)

¿Para qué sirve endpoint?

Cada Endpoint tiene una tarea única. Por ejemplo, en una tienda online podrías tener:

  • Un endpoint para ver productos: /productos
  • Un endpoint para pagar: /pagar
  • Un endpoint para crear cuenta: /registro

¿En que casos se usa endpoint?

se usa cada vez que una aplicación (el Frontend) necesita pedir, enviar, guardar o borrar información que no tiene a mano y que está guardada en un servidor (el Backend).

los casos mas comunes en los que se usa endpoint son:

ingresas en una aplicación. Pero no tiene guardada todo la information en tu celular o computador

  • El caso: La app toca el endpoint /películas-tendencia.
  • El resultado: El servidor le responde con la lista de portadas y nombres que ves en pantalla.

Se usa cuando creas algo que antes no existía.

  • El caso: Cuando te registras en una red social y llenas el formulario con tu nombre y correo, al darle al botón "Registrar", esos datos viajan a un endpoint llamado /crear-usuario.
  • El resultado: El servidor recibe tus datos y los guarda para siempre en su base de datos.

Se usa para procesos de "identificación".

  • El caso: Cuando escribes tu usuario y contraseña en Facebook. Esos datos van al endpoint /login.
  • El resultado: El servidor revisa si la contraseña es correcta. Si lo es, te deja pasar; si no, le dice al Frontend que muestre un error en rojo.

A veces, un programa necesita hablar con otro que no es suyo.

  • El caso: Una página de clima usa un endpoint de Google Maps para saber dónde estás exactamente.
  • El resultado: Google le "presta" la ubicación a través de ese endpoint para que la página te diga si va a llover en tu ciudad.

¿Qué son los métodos HTTP?

son verbos que indican la acción que se desea realizar sobre un recurso en un servidor web y asi mismo se es entregada la información de regreso.

Métodos HTTP principales para requests:

Método Descripción Id empotente Uso típico
GET Obtener/recuperar un recurso del servidor Sí Leer datos, cargar páginas, consultar información
POST Enviar datos al servidor para crear un nuevo recurso No Enviar formularios, crear registros, subir archivos
POST Enviar datos al servidor para crear un nuevo recurso No Enviar formularios, crear registros, subir archivos
PUT Actualizar completamente un recurso existente Sí Reemplazar datos completos de un recurso
PATCH Actualizar parcialmente un recurso existente No Modificar campos específicos sin alterar todo el recurso
DELETE Eliminar un recurso del servidor Sí Borrar registros, eliminar archivos
HEAD Obtener solo los headers (metadatos) de un recurso, sin el body Sí Verificar existencia, obtener metadatos (tamaño, fecha modificación)
OPTIONS Obtener los métodos HTTP permitidos para un recurso Sí CORS preflight requests, descubrir capacidades del endpoint
TRACE Depuración: devuelve la request recibida (eco) Sí Diagnóstico, debugging
