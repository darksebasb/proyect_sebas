¿Qué es git?

es un sistema de control de versiones o una agenda donde puedes mirar el historial de lo que haz hecho en ese proyecto y a su vez permite rastrear y gestionar los cambios realizados en archivos de código fuente durante el desarrollo de software.

¿Para qué sirve git?

  • Control de versiones: Guarda el historial completo de cambios en tu código
  • Trabajo colaborativo: Permite que varios desarrolladores trabajen simultáneamente sin conflictos
  • Gestión de ramas: Crea líneas de desarrollo paralelas (features, correcciones, experimentos)
  • Recuperación de cambios: Permite volver a versiones anteriores si algo sale mal
  • Respaldo del código: Cada copia del repositorio es un backup completo
  • Revisión de código: Facilita la revisión y aprobación de cambios antes de integrarlos

¿En qué casos se usa git

  • Proyectos personales (portafolio, prácticas)
  • Proyectos en equipo (startups, empresas, open source)
  • Aplicaciones web, móviles, de escritorio
  • Sistemas embebidos y IoT
  • Controlar versiones de archivos de configuración
  • Documentación técnica
  • Scripts de automatización
  • Trabajo remoto con equipos distribuidos
  • Code review antes de publicar cambios
  • Integración continua y despliegue (CI/CD)
